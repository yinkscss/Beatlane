Phase 0 — Foundation

Week 0–1

Exit: empty product shell deploys; game loop prototype playable locally; repo/CI green.

Engineering

  • Scaffold Vite + React + TS app; PixiJS board prototype (8×8, tray of 3, drag-drop, clear rows/cols).
  • Supabase project: Postgres schema stub (users, runs, purchases), Edge Functions hello-world.
  • Foundry repo for session-save receipt + Boast attestation contracts (compile + unit tests).
  • Vercel + GitHub Actions: lint/typecheck/deploy preview on PR.
  • Wire Sentry + PostHog stubs (env-gated).

Product / design

  • Freeze screen inventory from design-pack.html into implementation tickets.
  • Confirm cUSD price table in UI copy ($0.15 / $0.19 / $0.49→$0.79→$0.99).
Done when: Classic clear loop works in browser; CI deploys a preview URL; contracts test suite passes offline.

Phase 1 — Alpha

Weeks 1–6 · PRD

Exit: closed testers can play Classic, sign in with Magic, and buy Second Chance / Reshuffle / Undo on Celo testnet.

Core gameplay

  • Full Classic mode: polyomino set (incl. Square 9), combo scoring, blast VFX (single → mega → chain).
  • Game-over detection when no tray piece fits.
  • Blitz mode (timer) playable offline — no tournament payouts yet.
  • Daily mode scaffold (seeded RNG) — soft-launch for alpha cohort optional.

Session saves (monetization)

  • Second Chance sheet + escalating price within a run.
  • Reshuffle, Undo, Swap offers from toolbar.
  • Continue packs shop UI; inventory consumed on use.
  • Server-side purchase intent → testnet cUSD pay → store tx hash + grant entitlement (app writes + chain receipt).

Auth & infra

  • Magic.link email/social login; create/link Supabase user; show truncated address + cUSD balance.
  • Edge Functions: validate placement/score for a run (anti-cheat v0), grant continues.
  • Upstash: rate-limit purchase + score submit endpoints.
  • Supabase Storage for static art/SFX if not in-repo.

Out of Alpha

  • Mainnet money, Boast mint, tournaments, Season Pass, Goldsky.
Done when: 10+ internal testers complete paid testnet continues; no seed phrases shown; PostHog sees play → game-over → revive funnel.

Phase 2 — Closed Beta

Weeks 7–12 · PRD

Exit: invited players spend real cUSD on mainnet; Boast works; global Classic leaderboard ships. (PRD mentioned Goldsky — we defer to app receipts + polling per locked stack.)

Chain & economy

  • Audit-ready pass on payment/receipt + Boast contracts; third-party audit before mainnet (non-negotiable for funds).
  • Mainnet Celo deploy; real cUSD checkout for continues/packs.
  • Boast mint ($0.29 cUSD) + shareable proof page/card.
  • Spend caps / cooldowns UX (responsible spending).

Game & meta

  • Daily Puzzle live (fixed seed, streaks).
  • Global + local leaderboards via HTTP polling + Upstash hot cache.
  • Hardened server-authoritative scoring (recompute clears from placement log).
  • Economy tuning pass on revive curve from alpha data.

Ops

  • Closed beta waitlist / invite codes.
  • Sentry alert budget; PostHog dashboards for D1/D7 and payer %.
Done when: ≥1 real cUSD continue on mainnet in production; Boast link verifies on-chain; leaderboard updates within polling SLA; audit findings closed or accepted.

Phase 3 — Public Launch

Week 14 · PRD

Exit: public web URL; Blitz tournaments with entry fee + rake; Season Pass; MiniPay-aware acquisition push.

Compete

  • Tournament lobby, entry ($1–$10 cUSD), boost-free Blitz runs, automated payout contract, house rake 15–20%.
  • Results + claim flow to Magic wallet.

Retention / revenue

  • Season Pass ($2.99) — continue/reshuffle track only (no cosmetics).
  • Public marketing site / splash from design pack; Boast virality loops instrumented.
  • MiniPay / Celo co-marketing materials (deep link + “works with MiniPay” path).

Launch checklist

  • Status page / incident runbook; rollback for Edge Functions + contract pause switch if designed.
  • KPI baseline week: D1, payer %, ARPPU vs PRD targets.
Done when: public play without invite; first tournament completes end-to-end with payouts; Season Pass purchasable.

Phase 4 — Post-launch

Ongoing · PRD

Focus: tune economy; scale reliability; only add cosmetics if data demands; optional indexer upgrade.

  • Revive/reshuffle price experiments; pack mix optimization.
  • Optional Goldsky / Ponder if app-receipt indexing becomes a bottleneck.
  • Paymaster / gasless UX spike if Magic alone feels sticky.
  • Expand chains only if Celo liquidity/acquisition warrants.
  • Cosmetics / marketplace — explicitly gated on evidence players want them (PRD non-goal for v1).
